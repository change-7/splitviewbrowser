# SplitViewBrowser (macOS SwiftUI)

SwiftUI + `WKWebView` 기반의 macOS 네이티브 앱입니다. 여러 AI 웹앱(ChatGPT / Gemini / Perplexity / 사용자 추가 사이트)을 동시에 열어 비교해서 볼 수 있도록 만든 스플릿뷰 브라우저입니다.

## 현재 구현 상태 요약

- macOS 네이티브 앱 (`SwiftUI App lifecycle`)
- `WKWebView`를 `NSViewRepresentable`로 래핑해서 패널에 표시
- 패널 수 선택: `1 ~ 5`
- 배치 정책: `가로` 고정 (바둑판 배치 제거)
- 각 패널 서비스 개별 선택 가능 (중복 선택 허용)
- 기본 서비스
  - `ChatGPT`
  - `Gemini`
  - `Perplexity`
- 사용자 사이트 추가/삭제 가능 (설정에서 관리)
- 패널 상단 툴바
  - 서비스 선택
  - 분석 대상 패널 지정 버튼
  - 최신 답변 수집 버튼
  - Back / Forward / Reload / Home
- 패널 하단 자동복사 바
  - 각 패널마다 자동복사 체크박스 표시
  - 서비스별 자동복사 지원 상태 배지 표시
  - 명시적 `수집` 버튼(최신 답변 수집)
- 링크 클릭/새 창 요청은 인앱에서 열지 않고 macOS 기본 브라우저로 외부 열기
- 인앱 네비게이션은 기본적으로 취소 (외부 열기 정책)

## 창/레이아웃 동작 (현재 앱 기준)

- 패널 최소 기준 폭은 모바일 폭 기준으로 유지 (`430pt` 기반)
- 창이 넓어지면 패널 폭도 함께 넓어짐 (현재 구현 정책)
- 창이 좁아질 때는 현재 스플릿 구성보다 작아지지 않도록 최소 창 크기 동적 제한
- 패널 수를 줄이면 앱 창 최소 크기도 해당 구성에 맞게 줄어듦
- 가로 배치에서 창 크기에 맞춰 스플릿 영역이 함께 확장/축소됨
- 폭이 부족한 경우 스크롤로 탐색 가능 (`ScrollView`)

## 상단 UI (툴바)

- 패널 수 버튼(`1~5`)
- 프리셋 버튼 영역
  - `없음` 선택지 포함
  - 각 프리셋을 버튼으로 직접 선택
  - 선택된 프리셋만 강조 표시
  - `+` 버튼으로 현재 상태를 프리셋 저장
- 프롬프트 저장소 아이콘 버튼 (팝업)
- 설정 아이콘 버튼 (팝업)
- 답변 수집/분석 전송 영역
  - 분석 대상 패널 선택
  - 수집 개수 표시
  - 수집본 일괄 전송 버튼
  - 수집본 비우기 버튼

## 답변 수집 / 분석 패널 전송 (신규)

- 각 패널에서 사이트 자체 `복사` 버튼을 먼저 눌러 원하는 답변을 지정
- 사이트 자체 `복사` 버튼을 누르면 "방금 직접 복사한 답변"이 해당 패널 수집 버퍼에 자동 저장
- 패널 상단/하단의 `수집` 버튼은 마지막 직접 복사 답변을 수동으로 다시 수집할 때 사용
- 상단 툴바에서 `분석 대상 패널`을 지정
- `전송` 버튼으로 수집된 답변들을 하나의 비교/분석 프롬프트로 합쳐서 분석 대상 패널 입력창에 자동 입력 후 전송 시도
- 전송 버튼 탐지 실패 시에도 입력까지는 수행하고 상태 메시지로 안내
- 수집 데이터는 패널별로 1개씩 최신값으로 갱신 저장 (같은 패널에서 다시 수집하면 덮어씀)

참고:

- 답변 수집/직접 복사 감지는 웹 UI(DOM) 구조에 의존하므로 사이트 UI 구조 변경 시 정확도가 변할 수 있습니다.
- 현재는 텍스트 추출 중심이며, 스크린샷 fallback은 미구현입니다.

## 프리셋 기능 (현재 동작)

- 프리셋 저장/적용/삭제
- 이름 변경, 복제, 순서 변경, 잠금
- 프리셋에 저장되는 항목
  - 패널 수
  - 패널별 서비스 선택
  - 창 크기 (content size 기준)
- 활성 프리셋 선택 상태에서 아래 항목을 바꾸면 프리셋에 자동 반영
  - 패널 수
  - 패널별 서비스
  - 창 크기
- 잠금된 프리셋은 자동 갱신되지 않음

## 프롬프트 저장소 (팝업)

- 상단 아이콘 클릭 시 팝업 창 형태로 열림
- 저장된 프롬프트 관리
  - 저장 / 수정 / 삭제
  - 클릭 복사
  - 검색
  - 태그
  - 즐겨찾기
  - 정렬
- 팝업 상단 `X` 버튼으로 닫기 가능
- 팝업 바깥 영역 클릭 시 닫힘

## 설정 (팝업)

설정 화면은 좌측 카테고리 + 우측 상세 내용 구조입니다.

- `General`
- `Custom Sites`
- `Auto Copy`
- `Presets`
- `Backup / Restore`
- `Diagnostics`

특징:

- 우측 상세 영역은 스크롤 가능 (긴 설정 내용 잘림 방지)
- `Auto Copy`, `Presets` 항목은 내부 작은 스크롤 박스 없이 전체 폭 활용
- 팝업 상단 `X` 버튼으로 닫기 가능
- 팝업 바깥 영역 클릭 시 닫힘

## 자동복사 (현재 구현)

- 패널 하단 체크박스로 패널별 활성/비활성

## 빌드 / 실행 방법

### Xcode 실행

1. `SplitViewBrowser.xcodeproj` 열기
2. 스킴 `SplitViewBrowser` 선택
3. `My Mac` 대상으로 실행 (`⌘R`)

### CLI 빌드

```bash
xcodebuild -project SplitViewBrowser.xcodeproj \
  -scheme SplitViewBrowser \
  -destination 'platform=macOS' \
  build
```

### 테스트

```bash
xcodebuild -project SplitViewBrowser.xcodeproj \
  -scheme SplitViewBrowser \
  -destination 'platform=macOS' \
  test
```

### 실행용 `.app`

이 작업 폴더에는 빌드 후 복사된 실행용 앱 번들(`SplitViewBrowser.app`)을 둘 수 있습니다. 더블클릭으로 바로 실행 가능합니다.

### DMG 생성 (설치형)

```bash
./scripts/build_release_app.sh
./scripts/create_dmg.sh ./SplitViewBrowser.app ./SplitViewBrowser-Installer.dmg
```

Developer ID 서명 + 노터리제이션(자격 증명 필요):

```bash
export DEVELOPER_ID_APP='Developer ID Application: ...'
export NOTARY_KEYCHAIN_PROFILE='your-notary-profile'
./scripts/sign_and_notarize_dmg.sh
```

## 수동 테스트 체크리스트 (현재 기능 기준)

1. 상단 툴바에서 `1~5` 패널 전환 시 레이아웃과 최소 창 크기가 즉시 반영되는지 확인
2. `4~5` 패널에서도 가로 배치가 유지되고 폭 부족 시 가로 스크롤로 동작하는지 확인
3. 각 패널의 서비스 드롭다운 변경 시 해당 서비스 홈으로 이동하는지 확인
4. 패널 하단 자동복사 체크박스가 모든 패널에 표시되는지 확인
5. 패널 링크 클릭 시 앱 내부가 아니라 macOS 기본 브라우저에서 열리는지 확인
6. 설정 > `Custom Sites`에서 사이트 추가 후 패널 선택지에 즉시 나타나는지 확인
7. 설정 > `Auto Copy`에서 서비스 규칙 변경 시 패널 표시/동작이 반영되는지 확인
8. 프리셋 선택 후 패널 수/서비스를 변경했을 때 같은 프리셋에 자동 반영되는지 확인
9. 프리셋 선택 후 창 크기 변경 -> 다른 프리셋 갔다가 돌아오면 창 크기가 복원되는지 확인
10. 잠금 프리셋 선택 후 상태를 바꿔도 프리셋 값이 변경되지 않는지 확인
11. 프롬프트 저장소 팝업에서 저장/수정/복사/검색/태그/즐겨찾기가 동작하는지 확인
12. 설정/프롬프트 팝업 바깥을 클릭하면 팝업이 닫히는지 확인
13. 설정 > `Backup / Restore`에서 JSON 내보내기/가져오기 후 프리셋/프롬프트/사이트가 복원되는지 확인
14. 설정 > `Diagnostics`에서 로그 복사/초기화가 동작하는지 확인
15. 각 패널 상단 `수집` 버튼 클릭 시 수집 개수가 증가하고 수집 아이콘이 강조되는지 확인
16. 상단에서 분석 대상 패널 선택 후 `전송` 클릭 시 대상 패널 입력창에 비교 프롬프트가 입력/전송되는지 확인
17. 전송 실패 또는 입력창 미탐지 시 상단 상태 메시지로 안내되는지 확인

## 제약 / 알려진 이슈

- 일부 사이트 로그인/캡차/보안 정책(WebAuthn 포함)은 `WKWebView` 환경에서 동작이 다를 수 있습니다.
- 자동복사 기능은 사이트 DOM 구조에 의존하므로, 서비스 UI 변경 시 규칙 재조정이 필요할 수 있습니다.
- 답변 수집/자동전송 기능도 웹 UI(DOM) 구조에 의존하므로, 사이트 업데이트 시 일부 서비스에서 추출/전송 성공률이 낮아질 수 있습니다.
- 외부 사이트 정책 변경으로 특정 서비스의 화면 구조/동작이 달라질 수 있습니다.
- 링크는 의도적으로 외부 브라우저 열기 정책을 사용하므로, 인앱 링크 탐색은 제한됩니다.
- `scripts/sign_and_notarize_dmg.sh`는 Apple Developer 자격 증명(인증서/노터리 프로필)이 필요합니다.
